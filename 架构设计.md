# NXCopilot — 软件架构设计

> 基于 NXOpen Python API 开发 AI Agent 驱动的 NX CAD 助手。

## 核心设计理念

NXOpen Python 作为中间表示（仅供 LLM 分析，不用于重建）：

```
NX 模型 (.prt)
  → NXOpen 读取特征树
  → 转为 NXOpen Python 代码 (LLM 阅读)
  → LLM 分析 → 局部修改指令
  → NXOpen 执行修改
  → 更新后的模型
```

## 与 CATIA 版的关键差异

| 维度 | CATIA | NX |
|------|-------|-----|
| 连接层 | pywin32 COM | NXOpen.Session.GetSession() |
| 中间表示 | VBS 代码 | NXOpen Python 代码 |
| 参数修改 | FindObjectByName | Expression EditWithUnits |
| 特征创建 | MCP Server (87 tools) | NXOpen Builder API |
| 代码验证 | VBS 语法检查 | Python ast 模块 |
| 执行方式 | COM ExecuteScript | exec() 同进程 |

## 三大能力

### 正向设计
用户: "拉伸20mm" → LLM → NXOpen 创建代码 → 执行

### 逆向理解
用户: "分析结构" → 读特征树 → NXOpen 代码 → LLM 分析 → 回答

### 混合修改
用户: "孔径改12mm" → 读特征树 → LLM 定位 → Expression 修改 → 执行
